# Eval Mock-Mode Discipline

Governs `[[run-mcp-evals]]`, `[[forge-from-openapi]]`, and every MCP server generated by the Emdash forge.

- **Mock-mode** proves wire protocol and response-handling logic.
- **Live-mode** proves the API contract hasn't drifted.
- Neither replaces the other.

---

## When to use mock vs. live

### CI (push / PR / scheduled)

- Run `--mock-only` by default.
- Tests without `mock_response` are **skipped** (not failed); appear in NDJSON as `mode: "skip"`.
- Growing skip count = add mock coverage, not live keys to CI.

### Pre-release / manual verification

- Run `--live-only` or default hybrid before tagging.
- Catches field renames, schema changes, new required params.
- Requires API key via `get-secret <KEY_NAME>` per `rules/secret-provisioning.md`.

### High-confidence releases

- Run `--mock-only` first (fast gate), then `--live-only` (contract gate).
- Both must pass before publishing to the registry.

## Env-var hook: every forged MCP server MUST implement it

Any server generated by `forge-from-openapi` or `migrate-to-hardened` MUST include this interceptor in `src/utils/http.ts` (or inline in `src/index.ts`):

```typescript
// eval-mock-mode-discipline.md: env-var mock hook
// Set MCP_MOCK_RESPONSE_JSON=<base64-encoded JSON> to intercept all outgoing HTTP.
// Format: { status: number, body: unknown }
// Cleared per-process; re-spawn the server for each test in the eval harness.
function maybeGetMockResponse(): { status: number; body: unknown } | null {
  const raw = process.env.MCP_MOCK_RESPONSE_JSON;
  if (!raw) return null;
  try {
    return JSON.parse(Buffer.from(raw, 'base64').toString('utf8'));
  } catch {
    return null;
  }
}

async function httpFetch(url: string, init?: RequestInit): Promise<Response> {
  const mock = maybeGetMockResponse();
  if (mock) {
    return new Response(JSON.stringify(mock.body), {
      status: mock.status,
      headers: { 'Content-Type': 'application/json' },
    });
  }
  return fetch(url, init);
}
```

- All outgoing HTTP calls MUST route through `httpFetch`, never bare `fetch`.
- This is the only server-side requirement for Option A mock-mode.

## Anti-patterns (build-fail)

### Tests requiring live keys without tagging

- Any test missing `mock_response` and not tagged `"live_only": true` silently breaks CI.
- Fix: add `mock_response` OR tag explicitly:

```json
{
  "id": "charge-real-card",
  "live_only": true,
  "_comment": "Requires STRIPE_SECRET_KEY. Skipped in CI. Run manually pre-release."
}
```

- Tests tagged `live_only: true` are **skipped** (not failed) under `--mock-only`.

### Mock hiding schema mismatch

- Every `mock_response` MUST be sourced from a real API call or official docs.
- Include source in `_comment`:

```json
"mock_response": {
  "_source": "Resend API docs v1 — GET /emails/{id} — 200 response shape",
  "status": 200,
  "body": { "id": "...", "object": "email" }
}
```

### Bare `fetch` in the server

- Any direct `fetch(url, init)` call (not via `httpFetch`) cannot be mock-intercepted.
- Detect with:

```bash
grep -r "^\s*fetch(" src/ --include="*.ts" | grep -v httpFetch | grep -v "// bypass"
```

- Any hit is a drift violation per `[[drift-detection]]`.

### Committing Option B intercept scripts

- `/tmp/mcp-mock-intercept.mjs` is ephemeral — never track in git.
- Option B is a fallback for legacy servers; new servers always implement Option A.

## Adding mock coverage to existing tests

When `--ci` output shows `skip` entries:

1. Run live once: `--live-only --ci 2>/dev/null | jq .`
2. Copy the `actual` field from NDJSON output.
3. Paste as `mock_response.body`; set `mock_response.status` to the HTTP status.
4. Add `"_source": "observed from live run <date>"`.
5. Re-run `--mock-only` — test must now pass.

---

## Forge requirements

`forge-from-openapi --harden` MUST:

- Emit the `httpFetch` wrapper in `src/utils/http.ts`.
- Replace all bare `fetch` calls with `httpFetch`.
- Add `MCP_MOCK_RESPONSE_JSON` to the server README under "Environment variables".
- Emit at least one `mock_response` per golden test (using OpenAPI example response as source).

`migrate-to-hardened` MUST:

- Detect bare `fetch` calls and replace with `httpFetch`.
- Emit a migration note when adding the hook.

---

## CI gate policy

```yaml
- name: MCP Eval (mock-only)
  run: npx claude --headless "/run-mcp-evals --mock-only --ci"
  # Passes even if some tests are skipped (live_only).
  # Fails if any mock test fails or any server fails to spawn.
```

- Never add live-mode steps to CI without injecting the API key via a repository secret.
- Gate live-mode on `workflow_dispatch` or `release/**` — not on every push.
