# Image Sourcing + Generation Quality (***NON-NEGOTIABLE***)
Every image on every shipped page must be either (a) a real primary-sourced photograph downloaded into the repo OR (b) an AI-generated photograph crafted to be indistinguishable from a real, cinematic, gorgeously-lit one. Generic, flat, "good enough" AI output, unrelated stock photography, or hot-linked third-party assets without provenance are all rejected at build.
Priority order for every image slot: (1) Real photo from the institution's own archive, downloaded into `public/images/<area>/` with descriptive slug; (2) Real photo from Wikimedia Commons / Library of Congress / NPGallery / state historical society / verified press wire — DOWNLOAD into `public/images/<area>/` (never hot-link production-critical media — third-party CDNs disappear, rename files, change license terms); (3) AI-upscale a small but authoritative source (Real-ESRGAN, Gigapixel, Topaz Photo AI, or hosted equivalent) when the only available primary photo is sub-spec; (4) AI-generated hyper-realistic photograph crafted specifically for the slot (last resort for decorative surfaces only — NEVER on historical timelines per [[timeline-authenticity]]).
Web-sourcing cadence: for every new image slot, run a meticulous search BEFORE prompting any generator. Searches in order: Wikipedia article on the subject → Wikimedia Commons category → Library of Congress / NARA / NPGallery / Internet Archive book scans → state/local archives (NJ Historical Society, NYPL Digital Collections, Newark Public Library Charles F. Cummings NJ Information Center) → Flickr Commons (institutional uploads only — Smithsonian, LoC, BPL) → academic repositories (JSTOR-indexed scans) → press wire (with documented permission) → the institution's own Flickr/Instagram/blog (download with credit). Minimum 3 searches per slot before deciding nothing real exists. Save every candidate URL + Commons file page + author + license + capture-date into `public/images/<area>/_credits.json` so the next pass can audit provenance.
Download workflow: never `<img src="https://third-party-cdn/foo.jpg">` in shipped JSX. Curl/wget the image into `public/images/<area>/<slug>.{webp|avif|jpg}`, run through Sharp (`sharp input.jpg --resize 1600 --webp --quality 82 -o output.webp`) for an AVIF/WebP/JPEG triplet, log the source URL + author + license + date in `_credits.json` next to the file. Component renders `<img src="/images/<area>/<slug>.webp">` with companion `<picture>` AVIF source for browsers that support it. Build validator (`validate-image-provenance.mjs`) greps shipped JSX/TSX for any `src="https://"` image hot-link, fails the build, lists each violation with the local-mirror path it should use.
AI upscaling: when the only real photo is <1024px on the long edge but is authoritative (e.g., a digitized 19th-century engraving from Internet Archive at 600px), upscale with **Real-ESRGAN** (open source — `realesrgan-ncnn-vulkan -i in.jpg -o out.png -s 4 -n realesrgan-x4plus`) or **Topaz Photo AI** (paid, sharper for portraits with hair/skin) BEFORE shipping. Never ship a sub-800px hero. Tag the upscaled file's `_credits.json` entry with `upscaled: { tool: "Real-ESRGAN x4plus", from: 600, to: 2400 }` so the provenance is honest. Never alter the historical content of a photo via inpainting/outpainting — that crosses into fabrication.
DALL·E / GPT Image 1.5 / Sora prompt-craft (***whenever AI generation is the chosen path***):
  Step 1 — Imagine the IDEAL frame. Close your eyes, picture exactly what would belong in this slot for this audience: subject + composition + light + lens + emotion + era + palette. Write that frame out in 2-3 sentences of cinematographer's language BEFORE touching the model.
  Step 2 — Lock the medium. Default: "hyper-realistic editorial photograph, shot on a Leica SL3 with a 50mm Summilux f/1.4 lens, shallow depth of field, ISO 200, 1/250s, golden-hour natural side-lighting from a north-facing window, 35mm full-frame sensor look, color-graded warm midtones / desaturated shadows, fine 35mm-film grain, no motion blur, no posterization, no chromatic aberration, magazine-cover quality." Adjust lens/light for the moment but keep the realism contract intact.
  Step 3 — Render the subject as a single concrete sentence: "An [age] [role] [verb] [object] in [setting]" — no abstractions, no "and also". Add 1-3 sensory specifics: "steam curling from the bowl", "flour on her forearm", "a single bead of condensation on the pitcher".
  Step 4 — Banish AI tells: "no plastic skin, no waxy faces, no impossible anatomy, no hexagonal pupils, no extra fingers, no melted text, no duplicate features, no over-symmetrical composition, no center-framed mug-shot pose, no over-saturated color, no HDR-glow halos, no fake bokeh discs". Include the negative prompt block on every generation.
  Step 5 — Specify dimensions + crop: target 1600×1200 (4:3) or 1920×1080 (16:9) for hero; 1200×630 (1.91:1) for OG; 1080×1080 (1:1) for cards. State in the prompt.
  Step 6 — Critique-and-remix: every generation gets graded 1–10 by an AI vision pass (or human) against the imagined frame; <8 → remix with specific corrections (more side-light, less symmetry, warmer palette, real-skin texture). Maximum 3 critique rounds; if still <8, switch model or fall back to real photography.
Brand-consistency: every AI image generated for a single project shares a consistent lens, palette, and grading reference. Set those once in `public/images/_house_style.md` and pass into every prompt as a verbatim block ("HOUSE STYLE: Leica SL3 50mm Summilux f/1.4, natural golden-hour light, warm midtones / desaturated shadows, fine 35mm-film grain, magazine-editorial composition"). Inconsistent lenses/light/grading across images on one page = build fail in visual-QA review.
Quality bar (every image, AI or real, before shipping): hyper-realistic, cinematic, gorgeous, beautiful. If the picture would not look at home on a New Yorker spread, a National Geographic feature, a Pentagram annual report, or a Wieden+Kennedy campaign, the picture needs another pass. "Looks AI-generated" = ship-blocking. "Generic stock" = ship-blocking. "Flat phone snapshot" = ship-blocking unless it's the institution's own archival document being honored as-is.
Apply across: every hero, every section header, every blog post header, every team-grid avatar, every services-card image, every donate-page evidence shot, every annual-report cover, every press-kit asset, every OG card. Banned domains/sources without override: Unsplash (generic stock — never represents a real institution), Pexels (same), Shutterstock, Adobe Stock, iStock, Getty (without explicit licensed permission), AI generators for historical timelines (see [[timeline-authenticity]]).
