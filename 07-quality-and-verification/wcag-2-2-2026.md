# WCAG 2.2 / ADA Title II 2026 — Verification Gate

## Compliance Deadlines (current)

### DOJ Title II Web Rule (extended via Apr 2026 Interim Final Rule)

- **≥50K population entities** — April 26, 2027
- **<50K population / special districts** — April 26, 2028
- **Technical standard** — WCAG 2.1 Level AA

### Other mandates

- **HHS Section 504** — May 2026 for healthcare federal-fund recipients (parallel mandate)
- **WCAG 3.0** — Working Draft only (174 outcomes as of Mar 2026). Candidate Recommendation targeted Q4 2027, realistic Recommendation late-2029. **Target 2.2 AA, not 3.0.**

## WCAG 2.2 — 9 New Criteria

- **2.4.11 Focus Appearance** — axe-core auto: NO — manual: visual review at 6 breakpoints
- **2.4.12 Focus Not Obscured (Min)** — axe-core auto: NO — manual: scroll past sticky headers
- **2.4.13 Focus Not Obscured (Enhanced)** — axe-core auto: NO — manual: full focus visible
- **2.5.7 Dragging Movements** — axe-core auto: NO — manual: verify click / tap alternative
- **2.5.8 Target Size (Min 24px)** — axe-core auto: **YES** — axe + spot check
- **3.2.6 Consistent Help** — axe-core auto: NO — manual: contact / help in same location every page
- **3.3.6 Accessible Auth (Min)** — axe-core auto: NO — manual: no cognitive-test CAPTCHA, offer passkeys
- **3.3.7 Redundant Entry** — axe-core auto: NO — manual: never re-ask already-provided info
- **3.3.8 Accessible Authentication (Min)** — axe-core auto: NO — manual: same as 3.3.6
- **3.3.9 Accessible Authentication (Enhanced)** — axe-core auto: NO — manual: same plus more

**axe-core 4.10+ auto-tests only 2.5.8** — all 8 others require manual review or axe DevTools Pro. Add an explicit manual gate to the verification pyramid.

## Automated Gate (Playwright + axe-core)

```ts
import { test, expect } from '@playwright/test';
import AxeBuilder from '@axe-core/playwright';

const BREAKPOINTS = [375, 390, 768, 1024, 1280, 1920];

for (const width of BREAKPOINTS) {
  test(`a11y @ ${width}px`, async ({ page }) => {
    await page.setViewportSize({ width, height: 900 });
    await page.goto(process.env.PROD_URL!);
    const results = await new AxeBuilder({ page })
      .withTags(['wcag2a', 'wcag2aa', 'wcag21a', 'wcag21aa', 'wcag22aa'])
      .analyze();
    expect(results.violations).toEqual([]);
  });
}
```

## Manual Gate Checklist (per route)

- [ ] Tab through every interactive element; focus indicator visible (2.4.11)
- [ ] Scroll page with focus; sticky headers do not obscure focused element (2.4.12 / 13)
- [ ] Every drag / swipe has a click / tap equivalent (2.5.7)
- [ ] Help / contact link in same location every page (3.2.6)
- [ ] Forms do not re-ask info already provided in same session (3.3.7)
- [ ] CAPTCHA is invisible (Turnstile interaction-only) or passkey-friendly (3.3.8 / 9)
- [ ] Keyboard-only navigation completes all primary tasks (no mouse-only flows)
- [ ] Screen reader (VoiceOver Safari, NVDA Firefox) announces landmarks + headings + form labels correctly

## Lighthouse Gate

- Accessibility ≥95 (target 100)
- All `[role]` values valid
- Color contrast ≥4.5:1 body, ≥3:1 large / UI
- `<html lang>` set
- Every `<img>` has `alt` (empty for decorative, descriptive for content)
